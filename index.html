Core vim concepts

Philosophy

Very much unix-inspired: Small composable units for specific tasks, and the interface between them should be text.



This is key to why vim is so awesome at text editing but so terrible at everything else.

Vim is MODAL

  • Yes, key functions change depending on mode
  • Coders (or even just writers) don't spend all their time inserting text, why optimize solely for that?
  • 4 pseudo- "mode groups" to know:
    • Normal mode: Primary mode, operate on text. Delete, cut, paste, indent, etc.
    • Insert mode: Self-explanatory
    • Visual mode: Kind of like highlighting with the mouse
    • Ex mode: Like the command line. :w, :q, :s/foo/bar, etc.
  • (demo - mode-basics)

So what?

In normal mode, vim has a glut of motions and text objects that let you tell it what you want.

This is makes the arrow keys + mouse look like a joke.

Some example motions:

  • w - forward to next word
  • e - forward to end of word
  • 0 - first character of line
  • fc - forward to next occurrence of c
  • tc - forward 'til next occurence of c

Object selections

Vim has built-in notions of "words", "sentences", etc. Combine with movements for precise selections

  • iw - "inside word"
  • aw - "around word"
  • i( - "inside parentheses"
  • (other objects: sentences, paragraphs, {, etc...)

(demo - motions-objects)

Combining everything

The real power comes from combining these precise movements + selections with operators like delete, copy, etc, to create flexible "operations" on your text.

And even better, vim gives you the . command to repeat operations!

(demo - operator-motion-object)

Why this is life-changing

  • It's a language for describing what you want to do to your text; think: "change between quotes", "delete 3 words", etc
  • Way better than playing "hunt with mouse" or "hold down your arrow key"
  • (IMO), less chance of breaking flow
  • Learning a new "term" (new operator, new text object) dramatically increases the "sentences" you can compose
  • You can actually define your own motions/objects (e.g., ruby block) or operators (e.g., vim-surround)!

Learning Vim

Getting Started

  • Go vim or go home--Give it a real chance!
  • Yes, you will lose some productivity at first, but recovery should be short.
  • There are some easy things to make the transition smoother...
  • Grab a sensible (but not HUGE) vimrc template
  • Remap keys: caps->ctrl, and ESC->jj and jk are popular


Common wisdom: disable arrows in favor of hjkl, eschew use of the mouse, etc... IMO only if you're really masochistic.

Improving

  • One step at a time! Learning everything at once is impossible. Pick one thing, practice for a while, repeat
  • vim's :h is awesome! motion.txt, modes-intro, etc.
  • Watch the wizards for inspiration when feeling "stuck":
    • Derek Wyatt - http://derekwyatt.org/
    • Drew Neil - http://vimcasts.org/
    • Gary Bernhardt - ??? (all over the place)
  • But seriously... Practice! (See first bullet)

For non-vim users

(Yes, emacs folks, I'm talking to you...)

"Vim nature": injecting a vim-like philosophy/interface to other programs. It's becoming more and more common as people catch on to the awesomeness of vim.



  • emacs: evil mode - Oddly enough, best received of all...
  • sublime: vintage mode
  • eclipse: vrapper
  • set editing-mode vi in ~/.inputrc... or set -o vi
  • Firefox: vimperator, pentadactyl
